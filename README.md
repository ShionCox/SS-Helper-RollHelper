# SS-Helper [骰子助手]

`SS-Helper` 是一个面向 SillyTavern（酒馆）的事件骰子插件，用来把日常聊天式跑团扩展成更清晰的“事件出现 -> 掷骰判定 -> 结果结算 -> 剧情推进”流程。

它适合叙事向 TRPG、带检定的角色扮演聊天、单人冒险、剧情互动等场景。插件会在不明显打断对话体验的前提下，把技能、状态、事件分支、限时判定等机制接进酒馆聊天里。

## 主要功能

- 事件骰子系统：AI 生成事件后，插件会自动解析并生成可操作的事件卡片。
- 自动或手动掷骰：可以让 AI 决定某些事件自动结算，也可以全部由你手动点击掷骰。
- 技能系统：支持维护角色技能表，并把技能修正加入检定结果。
- 技能预设：可以保存多套技能配置，适配不同角色或不同世界观。
- 状态系统：支持受伤、恐慌、祝福、减益等状态，并自动影响后续检定。
- 状态编辑器：可以手动增删当前会话中的状态，便于剧情调整。
- 优势与劣势：支持优势/劣势判定。
- 爆骰机制：满足条件时可触发追加掷骰。
- 剧情分支：可为成功、失败、爆骰分别设置不同后果走向。
- 结果引导：掷骰后可自动给 AI 一条简短引导，让后续剧情衔接更自然。
- 摘要注入：自动把规则、轮次摘要、状态信息发给 AI，减少上下文丢失。
- 限时事件：支持倒计时判定，超时后可自动按失败结算。
- 聊天级数据保存：技能预设和状态会按聊天维度保存，不同会话互不混淆。

## 适合谁

- 想在酒馆里玩带检定的剧情互动
- 想让 AI 不只是“讲故事”，而是按照事件规则推进剧情
- 想保留跑团感，但又不想手动维护太多流程
- 想给角色增加技能、状态、后果分支和失败成本

## 安装方式

推荐使用 SillyTavern 的 Git URL 安装第三方扩展。

仓库地址：

`https://github.com/ShionCox/SillyTavern-SS-Helper`

常规安装步骤：

1. 打开 SillyTavern。
2. 进入扩展或插件管理页面。
3. 选择通过 Git URL 安装第三方扩展。
4. 粘贴上面的仓库地址并安装。
5. 安装完成后刷新酒馆页面。
6. 在扩展列表中启用 `SS-Helper [骰子助手]`。

如果你习惯手动安装，也可以把仓库文件放进酒馆的第三方扩展目录后重载页面。

## 快速开始

1. 安装并启用插件。
2. 在扩展设置中打开 `SS-Helper [骰子助手]` 面板。
3. 保持默认设置，或先按你的需求调整技能、状态、剧情分支、限时规则。
4. 建议开启“默认发送规则给 AI”，让模型更容易按插件协议输出事件。
5. 开始聊天后，当 AI 输出事件时，插件会自动生成事件列表卡片。
6. 你可以点击按钮手动掷骰，或让 AI 按设置自动处理。
7. 检定结果会回写到当前流程里，并继续推动剧情。

## 常用功能说明

### 1. 事件骰子

这是插件的核心。

当 AI 按插件规则输出事件后，插件会：

- 自动识别事件内容
- 在聊天中生成事件卡片
- 显示检定目标、骰式、DC、目标角色、技能修正等信息
- 允许你直接点击按钮完成掷骰

### 2. 技能系统

如果你希望角色能力长期参与判定，可以开启技能系统。

- 你可以在技能编辑器中维护技能名和数值
- 支持多套技能预设
- 支持导入、导出 JSON
- 当前聊天会保存所用技能状态

默认内置了一套通用叙事 TRPG 技能预设，适合作为起点使用。

### 3. 状态系统

状态系统适合处理持续性的剧情影响，例如：

- 受伤
- 惊吓
- 中毒
- 祝福
- 临时强化

状态可以设置为全局生效，也可以只对特定技能生效；还能设置持续轮次。后续事件检定时，插件会自动把这些修正计入结果。

### 4. 剧情分支

开启后，可以根据不同判定结果把剧情导向不同方向，例如：

- 成功时获得线索
- 失败时触发敌意或损失
- 爆骰时进入特殊剧情

这会让“掷骰结果”真正影响后续叙事，而不是只显示一个数字。

### 5. 摘要注入

插件会把最近几轮的重要事件、结果和状态整理成摘要，在发送消息前一起提供给 AI。这样做的好处是：

- AI 更容易记住当前局势
- 降低前后矛盾
- 减少重复解释
- 长对话里更稳定

### 6. 限时事件

如果启用限时控制，AI 可以给事件设置时间限制。你没在时间内处理的话，事件会按失败或超时规则自动结算，适合：

- 追逐
- 潜入
- 限时抉择
- 生存压力场景

## 常用命令

插件支持以下常见命令：

- `/roll 1d20`
- `/eventroll list`
- `/eventroll roll <事件ID>`
- `/eventroll help`

如果当前已有待处理事件，`/eventroll list` 会列出当前轮次的事件信息；`/eventroll roll <事件ID>` 可以直接对指定事件执行掷骰。

## 推荐设置

如果你是第一次使用，推荐先这样开：

- 启用事件骰子系统
- 默认发送规则给 AI
- 启用技能系统
- 启用状态系统
- 启用优势/劣势
- 启用爆骰
- 启用剧情走向分支
- 启用摘要包含走向文本
- 历史轮次数保持在 `3`

这套配置比较稳，既能体现规则感，也不容易把提示词堆得太重。

## 使用建议

- 首次使用时，先用默认技能预设跑一小段剧情，确认模型能稳定产出事件。
- 如果模型不太听规则，优先开启“默认发送规则给 AI”，并减少你自己的额外提示干扰。
- 如果你偏好完全可控的跑团感，可以关闭“允许 AI 决定自动/手动掷骰”。
- 如果你想让失败更有代价，建议同时开启状态系统、限时事件和剧情分支。

## 可能遇到的问题

### AI 没有触发事件

先检查：

- 插件是否已启用
- 事件骰子系统是否开启
- 是否启用了“默认发送规则给 AI”
- 当前模型是否愿意遵循结构化输出

### 技能或状态没有生效

先检查：

- 技能系统或状态系统是否开启
- 当前聊天下是否真的保存了对应数据
- 状态是否设置为对当前技能生效
- 状态是否已经过期或被禁用

### 事件卡片出现了，但不能掷骰

通常是因为：

- 该事件已经结算
- 当前轮次已经结束
- 事件被超时自动处理

## 说明

这是一个偏“使用体验增强”和“叙事跑团流程化”的插件，不是独立骰点平台，也不是传统群骰机器人。它更适合在 SillyTavern 里给角色扮演加入明确的判定结构和持续状态管理。
